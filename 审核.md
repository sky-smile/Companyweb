## 一、代码质量审查

### 1.1 代码风格一致性
- **server**: 有完善的 ESLint + Prettier 配置（`.eslintrc.cjs`、`.prettierrc`）
- **frontend/admin**: ⚠️ 缺少 linter/formatter 配置，代码风格靠人工维持

### 1.2 TypeScript 使用
- ✅ server 中 `any` 使用极少，仅出现在脚本文件 `fix-filenames.ts:43,66,71`
- ✅ admin/frontend 无 `any` 滥用，类型定义完善
- ⚠️ server ESLint 规则关闭了三项 `no-unsafe-*` 检查（`.eslintrc.cjs:22-24`），削弱类型安全保障

### 1.3 错误处理
- ✅ server 有全局异常过滤器 `all-exceptions.filter.ts`
- ✅ admin HTTP 层统一处理 401 跳转和消息提示
- ✅ frontend 提供错误边界 `error.tsx`

### 1.4 代码重复（突出问题）
- ⚠️ `NewsPage.tsx` 和 `ProductsPage.tsx` 结构高度相似（约 270 行），搜索/筛选/分类管理/Tabs 布局几乎一致

### 1.5 模块化程度
- ✅ server 模块化良好，每个模块有独立的 controller/service/repository/dto/interfaces
- ⚠️ admin 页面层职责过重，可进一步拆分

## 二、安全性审查

### 2.1 认证和授权 — ✅ 良好
- JWT 配置：要求 secret 最少 16 字符，Cookie 设置 `httpOnly`、`sameSite: lax`、`secure`(生产环境)
- Token 版本机制：每次请求校验 tokenVersion，支持登出/改密后立即失效
- RBAC 权限控制：`permissions.guard.ts` 实现细粒度权限检查
- 速率限制：ThrottlerGuard 配置 60 次/分钟
- 密码哈希：bcryptjs，salt rounds=10

### 2.2 敏感数据处理 — ✅ 良好
- `.env` 文件已在 `.gitignore` 排除
- 环境变量通过 Joi schema 严格校验

### 2.3 输入验证 — ✅ 良好
- 全局启用 `ValidationPipe`，配置 `whitelist`/`forbidNonWhitelisted`
- DTO 层使用 `class-validator` 装饰器

### 2.4 SQL 注入防护 — ✅ 良好
- 所有查询使用 TypeORM `createQueryBuilder` 或实体方法
- 无用户输入拼接

### 2.5 XSS 防护 — ⚠️ 需关注
- **前端**: `RichContent.tsx:11-28` 清理不够完善，未过滤 `<svg>`、`<math>` 等向量
- **后端**: `html-sanitizer.ts` 使用正则实现，建议迁移至成熟库（DOMPurify/sanitize-html）
- **admin 端**: `html-utils.ts` 清理过于简略

### 2.6 CORS 配置 — ⚠️ 需改进
- `main.ts:22-36` 硬编码 localhost 地址列表
- 生产环境需添加实际域名并使用 `https://`

### 2.7 依赖安全
- ✅ 未发现明显问题，建议定期运行 `pnpm audit`

### 安全风险总结

| 风险 | 级别 | 位置 |
|------|------|------|
| XSS 清理依赖正则，可能被绕过 | 中 | `server/src/common/utils/html-sanitizer.ts` |
| admin 端 HTML 清理过于简略 | 中 | `admin/src/lib/html-utils.ts` |
| CORS 生产域名需配置 | 低 | `server/src/main.ts:22-36` |
